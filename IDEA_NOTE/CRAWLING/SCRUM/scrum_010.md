### 05. 개발시 알야아 할 내용

1. GIT 관리 : : 
	- 운영계 PULL이던, 개발계 PULL 시작이던 중요하지 않음
	- 반영시 어차피 따로 PULL 받아야 하므로 둘중에 하나 최신 PULL 받고 시작
	
	1. 개발계(dev) - PR 
		- 임의 PR로 개발계는 자유롭게 반영 가능
	
	2. 운영계(prd) - PR 
		- 운영계 pull받고 다시 운영계에서 해당 개발 내용만 merge
		- PR요청 후, 담당자 검토이후 반영 (PR 담당자 : 팀장님, 재유 팀장님, 동혁님)
		- 완료된 이슈는 close
